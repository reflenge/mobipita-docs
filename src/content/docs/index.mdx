## クイックスタート

### アーキテクチャの理解

1. **[テナントモデル](/architecture/tenant-model)** - RootTenant、Tenant、Locationの関係性とパターン
2. **[Stripe Connect](/architecture/stripe-connect)** - 決済システムの設計と実装方針
3. **[データベース設計](/architecture/database-design)** - マルチテナントDBの設計パターン

### 技術スタック

- **フロントエンド**: Next.js
- **認証**: Clerk / Better Auth
- **データベース**: PostgreSQL (Neon/Supabase)
- **ストレージ**: Cloudflare R2 / AWS S3
- **決済**: Stripe Connect
- **インフラ**: Vercel → AWS (移行予定)

## プロジェクト構成

このプロジェクトは以下のような構成になっています：

- **Master App**: マスター管理画面（`main`ブランチ）
- **Customer App**: 顧客向けアプリケーション（`preview`, `development`, `dev-${feature-name}`ブランチ）

詳細は[アプリケーション構成](/meetings/app-architecture)を参照してください。
