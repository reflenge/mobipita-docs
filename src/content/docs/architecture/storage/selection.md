## 結論

> **Status: 採用（暫定）**  
> 更新日: 2025-12-22  
> **アプリ内アップロードは Convex File Storage を採用（暫定）**

## どう使う想定か

- クライアントは **Upload URL を発行**してからアップロードする
- DB には `storageId` などの参照とメタデータだけを保存する

## 境界（マルチテナント）

- ファイル参照を持つレコードにも `rootTenantId` を必ず保持する
- 閲覧URLの発行や参照の扱いは必ず権限チェック付きにする

## 併用・将来案

- 大容量やCDN要件が出たら **R2/S3 併用**を検討する
- その場合も `rootTenantId` で必ず論理分離する
